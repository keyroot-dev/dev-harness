# 外部依存の取り扱い

外部依存（ライブラリ・フレームワーク・外部 API・CLI ツール）の仕様に依拠して作業するときに適用される。調査の手順・情報源へのルーティング・出典の付け方の正本は `.claude/skills/tech-research/SKILL.md` — ここでは再定義せず、**いつ必ず使うか**だけを定める。

```
 外部依存に触れる作業
   │
   ├─ 新しい API 面・設定・バージョンに触れる ──▶ 書き始める前に Skill('tech-research') で裏取り
   │    ・外部依存の API・設定を新しく使う／使い方を変えるコードを書くとき
   │    ・依存を追加・更新するとき（バージョン選定はリリース情報で裏付ける）
   │    ・外部依存が出すエラー・想定外の挙動をデバッグするとき
   │
   └─ リポジトリ内で確立済みの利用パターンの踏襲 ──▶ 既存コードが真実の源（裏取り不要）
        （ただし踏襲のつもりが新しい API 面に触れたら、上の経路に戻る）
```

- 記憶で即答・即実装できそうな場合**こそ**裏取りする。古い API を確信を持って書くのが最も危険な失敗（理由の正本: `.claude/skills/tech-research/SKILL.md` 冒頭の「なぜ記憶で即答しないか」）
- 裏取りは「最新」ではなく**プロジェクトの実バージョンに一致した情報**に合わせる（手順はスキル側に定義済み）
