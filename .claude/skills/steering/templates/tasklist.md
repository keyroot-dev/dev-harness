# タスクリスト

## 🚨 タスク完遂のリマインダー

**このファイルの全タスクが `[x]` になるまで作業を継続すること。**

- 計画段階では「実装可能なタスク」のみを並べる（将来やるかも／検討中は含めない）
- **タスクの状態は3値で記録する**: `[ ]` 未着手 → `[-]` 実装中（着手時に付ける）→ `[x]` 完了（テストが緑になった時のみ付ける）。着手時に `[x]` を付けない
- **1タスク = 1つの振る舞い（= 1つのテストで駆動できる単位）**。「〇〇クラスを実装」ではなく「〇〇が××のとき△△を返す」の形で書く
- 各タスクは 🔴Red→🟢Green→🔵Refactor で実装する（テストを先に書く）
- スキップは技術的理由のみ。スキップ時は `- [x] ~~タスク名~~（理由: …）` の形式で明記する
- 大きすぎるタスク（複数の振る舞いを含む）は振る舞い単位のサブタスクに分割してから消化する

> 規律の詳細（タスク完遂・テスト駆動・スキップ可能な技術的理由・禁止事項・正しい完了の形）は
> **`steering` スキル モード2「タスク完遂規律（正本）」「テスト駆動規律（正本）」が単一の真実**。ここはその要約にすぎない。
