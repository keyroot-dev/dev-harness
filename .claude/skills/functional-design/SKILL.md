# 機能設計書作成スキル

このスキルは、機能設計書を**描写（description）として育てる**ための詳細ガイドです。

> **これは描写ドキュメントである（正本: `.claude/rules/document-management.md`）。**
> 機能設計書は、まだ存在しない機能を先取りして詳細に書くものではない。**実装した機能ぶんだけ、コードを真実の源として追記して育てる**。
> - `/setup-project` から呼ばれた初回は、Walking Skeleton で実装済みの縦切りのみを記述した**スタブ**（見出し構造＋実装済み分）を作る。
> - 以後は `/add-feature` が機能を実装するたびに、その機能の設計を追記する。
> - 未実装機能を憶測で詳述しない（context poisoning を避ける）。

## 前提条件

- `docs/product-requirements.md` (PRD) が存在すること（意図の参照元）。
- 追記対象の機能が**実装済み**であること（描写はコードから導出する）。スタブ作成時は Walking Skeleton の実装が真実の源。

## 既存ドキュメントの優先順位

**重要**: `docs/functional-design.md` に既存の機能設計書がある場合、
以下の優先順位に従ってください:

1. **既存の機能設計書 (`docs/functional-design.md`)** - 最優先
   - プロジェクト固有の設計が記載されている
   - このスキルのガイドより優先する

2. **このスキルのガイド** - 参考資料
   - 汎用的なテンプレートと例
   - 既存設計書がない場合、または補足として使用

**新規作成時**: このスキルのテンプレートとガイドを参照
**更新時**: 既存設計書の構造と内容を維持しながら更新

## 出力先

作成した機能設計書は以下に保存してください:

```
docs/functional-design.md
```
